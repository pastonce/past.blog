## C++指针

> * 指针是一个**存储内存地址的整数**，用于管理和控制内存
> * 指针的类型不影响指针的实质，而仅仅是假设该地址指向的数据的类型

### 指针的定义与使用

C++中使用`*`来声明指针变量，使用取地址运算符`&`来获取一个变量的内存地址，还可以使用`*`来解引用，获取指针地址指向的内存的数据。

```c++
#include <iostream>

int main() {
	void* ptr = nullptr;	// 初始化空指针
    int num = 10;
    
    ptr = &num;    			// 可以接受任何类型变量的地址
    *((int*)ptr) = 12;		// 使用前必须强制类型转换
    
    std::cout << num << std::endl;	// 12
}
```

* 未经初始化的指针（如`int* p`）是一个野指针，它指向一个未知的内存地址，直接解引用会导致未定义行为，**一般使用`nullptr`初始化空指针**
* `void*`定义的指针称为无类型指针或通用指针，它可以接受任何类型变量的地址，无需强制类型转换；**不能直接解引用`void*`定义的指针**，使用前必须强制类型转换为具体的数据类型指针
* 指向数组等较大跨度的数据时，指针默认存储着该数据块的首地址

```c++
int main() {
	char* buffer = new char[8];		// buffer指向8个char的首地址
	memset(buffer, 0, 8);
	
	delete[] buffer;  			   // 删除new申请的堆内存
}
```

### 多级指针

因为指针本身也是一种存储在内存某一位置的变量，所以同样可以被其他指针存储地址。多级指针就是指向指针的指针，存储着低级指针的地址。

```c++
#include <iostream>

int main() {    
	int num = 100;
    int* pnum = &num;	// 一级指针
    int** ppn = &pnum;  // 二级指针

    std::cout << "num = " << num << std::endl;
    std::cout << "*pnum = " << *pnum << std::endl;
    std::cout << "**ppn = " << **ppn << std::endl; // 解引用两次
}
```

## C++引用

> * 引用是在指针上的语法糖，本质相同
> * 引用创建的只是一个别名而**不是一个真正的变量**，它和它引用的变量共用同一块内存空间

### 引用的定义与使用

C++中使用`&`来声明引用，创建的只是已有变量的一个别名。**定义了一个引用后，不能改变引用的指向**；正因为如此，每个引用在定义时必须初始化（类似于一个指针常量）。

```c++
#include <iostream>

int main() {
    int a = 10;
    int b = 12;
    int& ref = a;	// ref是变量a的别称
    
    ref = b;   		// 并不代表ref改为引用b，而是代表将变量b的值赋予变量a
    std::cout << a << std::endl;	// 12
}
```

引用可以实现的功能都可以通过指针实现，但是引用通常会使代码更简洁易懂，使用也更方便安全。如引用通常用于函数传参：

```c++
#include <iostream>
#define Log(x) std::cout << "a = " << x << std::endl;

void increment1(int x) {	// 仅仅将变量值传入，局部变量的改变不影响传入变量
    x++;
}
void increment2(int* x) {	// 指针实现
    (*x)++;
}
void increment3(int& x) {	// 引用实现
    x++;
}

int main() {
    int a = 5;

    increment1(a);
    Log(a);		// a = 5
    increment2(&a);	
    Log(a);		// a = 6
    increment3(a);
    Log(a);		// a = 7
}
```

---

引用本身也可以被引用，但没有多级引用的概念，因为它们都是原变量的多个别名，即**同一变量可以有多个引用**。

**除了改变不了指向，引用比指针方便太多了，好用爱用多用能用就用*
