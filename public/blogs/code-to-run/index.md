## 一个简单的Hello World程序

```c++
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    std::cin.get();
    
    return 0;
}
```

`include`语句是预处理语句，编译器接受到源文件之后会先处理这些语句，即在实际编译之前进行处理

* 编译器会找到所有列出的头文件，并将其文件内容复制到该源文件中，所以直接手动复制也是可以运行的
* 上述代码中的 `cout`、`endl`、`cin` 等函数就定义在 `iostream` 头文件中
---
`main`函数是程序的入口，计算机从入口处逐行执行代码

* `main`函数虽然使用 `int` 作为返回值类型，但不一定需要返回值，不返回时默认返回了0
---
上述代码中的 `<<` 符号实际上是左移运算符的重载，可以理解为一个函数，其效果类似于`print`

* 运算符重载：对已有的运算符重新进行定义，赋予一种功能，以适应不同的数据类型。

* 定义重载的运算符的格式：

```c++
返回类型 operator 运算符（参数列表）
{
    函数体;
}
```

* 重载后运算符的优先级和结合性应该保持不变，也不能改变操作数个数（不能有默认参数）与语法结构

* 不可重载：类关系运算符`:`、成员指针运算符`*`、作用域运算 符`::`、`sizeof`运算符、三目运算符`? :`

## C++从代码到实际运行的流程

### 1.预处理

* 处理`#include`：查找并包含头文件到当前代码文件中
* 处理`#define`：对当前文件中所有用到宏的地方进行替换
* 删除注释
* .cpp $\rightarrow$ .i

### 2.编译

* 对代码进行语法检查
* 生成汇编代码
* .i $\rightarrow$ .s

### 3.汇编

* 翻译为可重定位目标文件
* .s $\rightarrow$ .o/.obj

### 4.链接

* 合并多个目标文件
* 解决函数调用关系
* 生成最终的可执行文件.exe

### 5.执行

* 调用操作系统中的加载器函数，将可执行文件中的代码和数据复制到内存
* 控制转移到程序的开头，开始运行

## 函数的声明与定义

为了代码的易读、复用和可维护性，一般不会将所有代码放在同一文件中，而是按模块拆分为若干子文件。比如可以将上述`Hello World`代码分为独立的两个文件如下：

```c++
// main.cpp
#include <iostream>

void Log(const char* message);

int main() {
    Log("Hello World!");
    std::cin.get();
    
    return 0;
}
```

```c++
// Log.cpp
#include <iostream>

void Log(const char* message) {
	std::cout << message << std::endl;
}
```

在同时编译运行时，编译器会独立地对两个文件进行编译，并由链接器将其中的函数调用逻辑（如上的`Log`函数）进行承接，其中就涉及到函数的声明与定义。

### 1.函数声明

形式上参照 `main.cpp` 中的`line 4`，主要包括函数名、传入参数的类型和数量、返回值类型，此处传入参数名称可以选择省略不写。声明的作用是告诉编译器该函数是存在的，但在编译时编译器并非确定声明的函数真的存在（相当于听其他人说有人叫这个名字，但并非真的确定有这样一个人），因此即使没有 `Log.cpp` 文件仍然能成功编译。

### 2.函数定义

函数的定义实际上包含了函数声明以及函数本身的内容（毕竟不声明就不知道内容属于哪个函数），即从头到脚塑造出了这个人。在链接时链接器会处理多文件的函数调用关系，确定 `Log` 函数真实存在且可用，否则无法生成可执行文件。
