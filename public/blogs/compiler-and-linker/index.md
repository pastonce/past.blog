## C++编译器

> * C++ 编译器获取用 C++ 编写的高级源代码，并将其转换为可执行的低级二进制机器代码
> * 除了翻译之外，编译器也可以优化 C++ 代码的性能、检测错误以及确保不同系统之间的兼容性

### 预处理

在编译之前，编译器会率先处理各个预处理命令，它们包括：`#include`、`#define`、`#if`/`ifdef`/`ifndef`/`endif`、`#pragma`.

* 对于`define`定义的宏，编译器将在代码中进行搜索并直接替换为相应的值或代码片段；
* 对于`include`引用的头文件，编译器则会将其指定的任何文件都简单地合并（近似于复制粘贴）到源代码中；
* `#if`/`ifdef`/`ifndef`/`endif`称为条件编译指令，它们依据特定的条件来指示预处理器判断保留或删除某段源代码，判断逻辑与`if`/`else`类似。其中`#if(n)def`相当于`if (not) defined`，可用于测试某个宏是否已被定义。在预处理阶段结束时，没有被预处理器保留的组会从程序中全部删除，不会出现在后续流程中。

---

此外，编译器还会在此阶段删除代码中的单行与多行注释，仅保留代码的功能组件；源代码中的字符和字符串也会被进一步处理，使其可执行，例如解析转义序列，并将字符转换为最适合执行环境的适当格式。

### 汇编

在此期间编译器会检查代码是否符合 C++ 语法规则和逻辑一致性。这涉及构建抽象语法树（Abstract Syntax Tree，AST）来验证结构正确性，并执行语义检查以捕获类型不匹配、未声明的变量或无效函数调用等问题。

语法检查无误后，编译器转向代码生成，将代码转换为特定于平台的二进制目标文件，其中包含机器可读代码以及所需数据，将用作链接过程的输入。

## C++链接器

> * C++链接器找到每个符号和函数的位置并将它们链接在一起，将多个汇编得到的目标文件合并为最终的可执行二进制文件。
> * 程序的入口点一般是`main`函数，**也可以自定义**，但必须要有一个入口点

### 可重定位目标文件

汇编成功之后，每个源代码文件都会得到一个可重定位目标文件，其中至少包含以下三个表：

* 未解决符号表：列出了本单元有引用但是不在本单元定义的符号及其出现的地址
* 导出符号表：提供了本单元具有定义，并且可以提供给其他单元使用的符号及其出现的地址
* 地址重定向表：提供了本单元所有对自身地址的引用记录

除此之外，可重定位目标文件中还会包含编译好的机器码、变量信息、字符串表等信息。

*可参考《深入理解计算机系统（第三版）》p467*

### 链接器工作顺序

当链接器进行链接的时候，首先决定各个目标文件在最终可执行文件里的位置。然后访问所有目标文件的地址重定义表，对其中记录的地址进行重定向（加上一个偏移量，即该编译单元在可执行文件上的起始地址）。然后遍历所有目标文件的未解决符号表，并且在所有的导出符号表里查找匹配的符号，并在未解决符号表中所记录的位置上填写实现地址。最后把所有的目标文件的内容写在各自的位置上，再做一些另外的工作，就生成一个可执行文件。

### C++提供的相关特性及注意事项

* `extern`：表示被修饰的变量或函数在其他编译单元中定义，即把该符号放入未解决符号表中（外部链接）
* `static`：表示被修饰的变量或函数不会被导出，只在当前单元中有效，对其他单元不可见，即不放入导出符号表中（内部链接）
* 对于任意的函数和变量，默认为外部链接，对于`const`修饰的变量，默认内部链接；外部链接的符号在整个程序范围中可用，这要求其他单元不能导出相同的符号；内部链接的符号则允许在不同的编译单元内有相同的名称，因为内部链接符号对其他单元不可见
* 头文件中一般仅写声明不写定义，因为头文件可被多个编译单元包含，存在外部链接的符号的定义时，极有可能导致链接器报错（solution：`static`/`inline`修饰；仅留下声明，将定义下放到另外一个单元中）
